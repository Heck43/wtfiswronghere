# 12 challenge

## описание ошибки
в конструкторе класса `FizzBuzz` переменная `self.num2` инициализировалась значением `4` вместо `5`. это приводило к тому, что кратность проверялась для чисел 3 и 4, выводя `buzz` для кратных 4 (например, 4, 8) и `fizzbuzz` для кратных 12.

## решение
изменили значение `self.num2` в конструкторе класса с `4` на `5`, чтобы восстановить стандартную логику fizzbuzz.
теперь программа работает корректно.
