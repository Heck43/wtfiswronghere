# wtfiswronghere — исправленный fizzbuzz

## что сделано

мы прошли все 13 испытаний и исправили все ошибки, от простых синтаксических опечаток до более сложных логических и объектно-ориентированных багов. для каждого испытания был изменен код и написан отчет о проделанной работе в файлах ридми внутри папок челленджей.

### список челленджей и исправлений:
* **01_challenge**: функция `fizzbuzz(max_num)` реализует базовый алгоритм fizzbuzz для диапазона от 1 до `max_num` с использованием локальных переменных делителей. была исправлена синтаксическая ошибка оператора присваивания `=` на сравнение `==` в проверке кратности.
* **02_challenge**: функция `fizzbuzz(max_num)` вычисляет fizzbuzz до переданного предела. была исправлена ошибка вызова функции `fizzbuzz()` без обязательного позиционного аргумента в точке входа.
* **03_challenge**: функция `fizzbuzz(max_num)` выполняет классический fizzbuzz, проходя по циклу. была исправлена ошибка с передачей строкового аргумента `'16'` вместо целого числа, что ломало функцию `range()`.
* **04_challenge**: функция `fizzbuzz(max_num)` выводит результаты fizzbuzz в консоль. была исправлена синтаксическая ошибка неверного отступа (`IndentationError`) перед вызовом `print`.
* **05_challenge**: функция `fizzbuzz(max_num)` открывает текстовый файл `myfile.txt`, читает из него значения делителей и лимита, а затем выполняет fizzbuzz. была исправлена ошибка с неверным именем файла (`mifile.txt` вместо `myfile.txt`) и устаревший синтаксис `print` из python 2.
* **06_challenge**: функция `fizzbuzz(max_num)` считывает значения делителей из импортированного конфигурационного модуля `conf`. была исправлена ошибка обращения к несуществующему атрибуту `conf.num` вместо `conf.num2`.
* **07_challenge**: функция `fizzbuzz(max_num)` запускает fizzbuzz до верхнего предела. была исправлена логическая off-by-one ошибка, когда вызов `fizzbuzz(99)` исключал число 99 из диапазона.
* **08_challenge**: класс `Fizz_Buzz` содержит метод `fizzbuzz(max_num)` для вывода последовательности. была исправлена ошибка неверного определения метода без параметра экземпляра `self`.
* **09_challenge**: функция `fizzbuzz(max_num)` выполняет fizzbuzz в заданном диапазоне. была исправлена логическая ошибка, из-за которой цикл `range(6, max_num)` начинался с 6 вместо 1, пропуская первые числа.
* **10_challenge**: скрипт импортирует внешний модуль `fizzbuzz` и вызывает функцию `fizzbuzz(max_num)` из него. была исправлена ошибка `TypeError`, возникающая из-за попытки прямого вызова самого модуля вместо функции внутри него.
* **11_challenge**: функция `fizzbuzz(max_num)` реализует fizzbuzz с проверками условий деления. была исправлена логическая ошибка неверного порядка условий, когда общая проверка на 3 и 5 стояла в конце и никогда не срабатывала.
* **12_challenge**: класс `FizzBuzz` инициализирует делители в конструкторе и предоставляет метод `fizzbuzz(max_num)` для вывода. была исправлена логическая ошибка, когда в конструкторе делитель `self.num2` задавался как 4 вместо 5.
* **13_challenge**: функция `fizzbuzz(max_num)` реализует стандартный fizzbuzz. были исправлены три синтаксические ошибки: незакрытая кавычка в строке, пропущенное двоеточие в объявлении цикла и опечатка в названии вызываемой функции.

## как исправляли

1. запускали скрипт из папки челленджа и смотрели на сообщение об ошибке.
2. искали проблемное место в коде по номеру строки из стектрейса.
3. анализировали причину (неправильный синтаксис, неверный тип данных, опечатки, логические ошибки).
4. исправляли баг в коде и добавляли милые комментарии.
5. документировали ошибку и ее решение в локальном readme-файле каждого челленджа.

## как тестировали

все тесты проводились вручную для каждого челленджа:
1. запускали исправленный скрипт с помощью команды `python XX_challenge.py` (где `XX` — номер челленджа от 01 до 13).
2. проверяли, что программа не выдает ошибок в терминале и успешно завершает работу.
3. сверяли вывод программы с эталонным выводом fizzbuzz:
   - для чисел, кратных 3, выводится `fizz`
   - для чисел, кратных 5, выводится `buzz`
   - для чисел, кратных и 3, и 5, выводится `fizzbuzz`
   - проверялись все числа в диапазоне от 1 до 99 включительно.
